# PR Commit

이슈 번호와 이슈 제목을 기반으로 변경사항을 분석하여 커밋 메시지를 생성하고 커밋을 수행한다.

## 인자

`$ARGUMENTS` 형식: `<이슈번호> <이슈제목>`

예시:
- `42 로그인 버그 수정`
- `123 사용자 프로필 페이지 추가`

## 수행 절차

1. `$ARGUMENTS`에서 첫 번째 토큰을 이슈 번호, 나머지 텍스트를 이슈 제목으로 파싱한다.

2. `git diff --staged`와 `git status`로 현재 상태를 확인한다.

3. staged 변경사항이 없으면 `git add -A`로 모든 변경사항을 스테이징한다.

4. 변경사항 전체를 분석하여 핵심 내용을 파악한다. 무엇이 추가/수정/삭제되었는지 기능 단위로 이해한다.

5. 아래 포맷으로 커밋 메시지를 작성한다:

```
[#이슈번호] 이슈제목
- 변경사항 요약
- 변경사항 요약
```

6. 작성한 커밋 메시지를 사용자에게 보여주고 확인을 받는다.

7. 확인 후 `git commit -m "..."` 으로 커밋을 실행한다.

## 커밋 메시지 작성 기준

- 첫 번째 라인은 반드시 `[#이슈번호] 이슈제목` 형식 (변경 금지)
- bullet point는 3~5개, 각 항목은 80자 이내로 간결하게
- "무엇을 했는가"에 집중하여 작성 (HOW가 아닌 WHAT)
- Co-authored-by 태그를 포함하지 않는다

## 예시

**입력:** `42 로그인 버그 수정`

**커밋 메시지:**
```
[#42] 로그인 버그 수정
- JWT 토큰 만료 시 자동 로그아웃 처리 추가
- 로그인 실패 시 에러 메시지 표시 개선
- 세션 유효성 검사 로직 수정
```
